一条"人提需求 → AI 批量造 → 机器自动验收"的流水线A pipeline: plain request → AI mass-generates → machine auto-verifies

已公开 · 逻辑全透明Public · logic fully transparent

我想解决的是这个问题：一句人话（"做个更紧张刺激的版本"）到底怎么变成机器能验收的东西。我的流水线把它拆成三层——人写假设、AI 翻成参数和可测目标、机器跑 200 局自动验收；达标放行，不达标自动打回重做甚至直接淘汰。有个偷懒的"轻松版"就是这么被门禁判为太闷又太拖、当场淘汰的，不靠人肉试玩。它能批量生成、批量验收 A/B 变体。我拿一个方块消除游戏当试验载体，所以成品顺带能玩——但值钱的是那条闭环，不是游戏。

The problem I wanted to crack: how does a plain-English wish ("make it feel more tense") actually become something a machine can check? My pipeline splits it into three layers — a human writes the hypothesis, an AI turns it into parameters and measurable targets, and a harness plays 200 rounds to verify. Pass and it ships; fail and it's auto-sent back or dropped. One lazy "relaxed" variant got auto-killed this way — the gate judged it too dull and too slow, no human playtesting needed. It generates and verifies A/B variants in batch. I used a block-puzzle game as the test payload, so the output happens to be playable — but the value is the loop, not the game.

企业级"自动派单"智能体An enterprise "auto-dispatch" agent

代码公开 · 可现场演示 / 录屏Code public · live demo / screencast on request

故障告警进来，这个 AI 自动判断是什么问题、该派给谁；遇到高风险的事强制转人工审批，不让它擅自动手。我真正下功夫的不是功能多，而是一个 AI 接进真实系统时该有的"笼子"：动作白名单、核心目标强制人工、效果用 eval 卡线（路由准确率低于 0.95 直接红灯）、每一步都能追溯。这条思路是从我做企业安全运营（SOC）的经验里长出来的。

An alert comes in; the agent works out what it is and who to route it to — and anything high-risk is forced through human approval, never auto-executed. What I actually invested in isn't feature count, it's the cage an AI needs when it touches a real system: an action allow-list, mandatory human sign-off on core targets, results held to an eval gate (routing accuracy below 0.95 fails the build), every step traceable. The approach grew out of my work building a security operations (SOC) system.

我给自己造了一套 AI 操作系统：把跨项目的经验、方法、决策结构化存起来，人和 AI 都能调用，还能在已有知识上推导出新结论。它每天在帮我自己干活——不是个摆设，这页背后的判断也有它的份。最近开始把里面通用的零件拆出来单独开源，第一件是给 Claude Code 做"规则按需加载"的 hooks 工具包。

I built myself an "OS" for working with AI: it stores experience, methods and decisions from across my projects in a structured way that both I and an AI can draw on, and it can derive new conclusions from what's already there. It runs my own days — not a showpiece. I've started open-sourcing its generic parts — first out: a hooks kit that gives Claude Code load-on-demand rules.
