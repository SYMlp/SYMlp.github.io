一条自动生成、自动验收软件的流水线A pipeline that generates and verifies software automatically

已公开 · 成品在线可玩Public · results playable online

输入一句需求描述，AI 会生成一个完整的软件新版本，再由程序自动验收：跑 200 局模拟测试，达标的放行，不达标的打回重做，还不行就丢弃，中间不需要人参与。我用一个方块小游戏做试验对象，验收合格的版本挂在网页上，可以直接玩到。做这个是想验证一件事：AI 产出的质量，可以不靠人盯，靠机器把关。

You type one sentence describing what you want. The AI generates a complete new version of the software, and a program then checks it: 200 simulated rounds, passing versions go through, failing ones get sent back for rework or discarded, with no human in between. I used a small block puzzle game as the test subject; the versions that passed are on the web and playable. I built this to test one idea: the quality of AI output can be guarded by machines rather than by people watching over it.

一个智能体接入企业系统的完整样板A working model of an AI agent wired into enterprise systems

研究项目 · 代码公开Research project · code public

这个项目没有实际上线。我做它是为了把一个问题彻底搞清楚：让 AI 接进企业系统干活，要配齐哪些东西才算稳妥。样板里这些都是真实做出来的：告警进来，它自动判断是什么问题、该派给谁；高风险操作强制停下来等人批准；每一步留有记录；判断准确率有自动化测试把关，低于 95% 不允许更新。这些标准不是凭空定的，来自我之前给企业建安全运营系统（SOC）的经验。

This one is not deployed in production. I built it to settle one question thoroughly: what does an AI agent need around it before it can safely work inside an enterprise system? Everything in the model is actually implemented: an alert comes in, it works out what the problem is and who should handle it; high-risk actions stop and wait for human approval; every step is logged; its accuracy is held by automated tests, and below 95% no update is allowed through. The standards aren't invented — they come from my earlier work building an enterprise security operations (SOC) system.

每天自用 · 代码公开In daily use · code public

我把各个项目里的经验、方法和做过的决策，整理成 AI 能直接读取的结构，让它跨项目、跨会话地记住这些，用得越久越顺手。这套系统我每天在用。其中通用的部分，拆成了四个开源工具发布。

I organize the experience, methods and decisions from my projects into a structure an AI can read directly, so it remembers them across projects and sessions, and gets more useful the longer it's used. I use this system every day. Its generic parts are published as four open-source tools.
